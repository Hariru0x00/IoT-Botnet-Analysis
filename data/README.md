# 1. data/ - Raw and Processed Data Repository

**Purpose:** Serves as the central storage hub for all data related to malware samples, both in their original form and after processing.

## Subdirectories

| Directory | Purpose | Example Content |
| :-------- | :------ | :------ |
| **raw/** | Contains original malware samples that have been renamed (e.g., with a `.bin` extension) to prevent accidental execution. | `b9d418...73.bin`, `israel.mips.bin` |
| **processed/** | Stores the results generated by analysis tools (static/dynamic). | JSON files from static analysis, PCAP files, CSV files containing API call logs. |
| **reconstructed/** | Stores reconstructed source code, such as decompiled C code and annotated assembly code. | `*_decompiled.c`, `NOTES.md`, `iocs.txt` |
